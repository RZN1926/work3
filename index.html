git workflow — принцип работы с ветками в гите, помогающий управлять изменениями в коде. Создание веток, слияние изменений, управление версиями кода и тд., что помогает команде нормально работать над проектом, минимизируя конфликты и улучшая качество кода одновременно


git rebase - обьединяет изменения


отличия от merge - merge создает коммит изменения, rebase меняет свою историю


git fetch - загружает коммиты, репки из удаленной репки в основную репку


отличия от pull - fetch загружает содержимое ничего не меняя, pull меняет все
